# Time Tracker v0.1 — Technology & Code Knowledge Base

---

## Architecture

```
Browser (React)  ──HTTP──►  Spring Boot (Port 8080)  ──JDBC──►  MySQL
     Port 3000                   REST API                       timetracker DB
```

**3-Tier Architecture:**
- **Presentation** — React (UI)
- **Application** — Spring Boot (business logic + REST API)
- **Data** — MySQL (persistence)

---

## Backend — Spring Boot

### Layer Structure

```
controller/   ← receives HTTP requests, sends HTTP responses
service/      ← business logic
repository/   ← talks to the database
model/        ← Java classes that map to DB tables
config/       ← security, error handling
```

Each layer only talks to the one below it. Controllers never touch repositories directly.

| Dependency | Purpose |

### application.properties

```properties
spring.datasource.url=${DB_URL:jdbc:mysql://localhost:3306/timetracker?...}
spring.datasource.username=${DB_USERNAME:root}
spring.datasource.password=${DB_PASSWORD:}
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=false
spring.jpa.open-in-view=false
```

- `${VAR:default}` — reads from environment variable, falls back to default. Keeps credentials out of code.
- `ddl-auto=update` — Hibernate auto-creates/updates MySQL tables from Java classes. No manual SQL DDL needed.
- `open-in-view=false` — prevents DB sessions from staying open during HTTP response rendering (performance best practice).

### model/Task.java

```java
@Entity              // map this class to a DB table
@Table(name="tasks") // table name in MySQL

@Id
@GeneratedValue(strategy = GenerationType.IDENTITY) // auto-increment primary key

@NotBlank            // validation: reject empty/null strings
@NotNull             // validation: reject null values
@Enumerated(EnumType.STRING) // store "TODO" not 0/1/2 in the DB column

@JsonIgnore          // exclude timeLogs from JSON (prevents infinite recursion)
@OneToMany(mappedBy = "task", cascade = CascadeType.ALL, orphanRemoval = true)
// One Task → many TimeLogs
// cascade=ALL: delete/save operations propagate to child TimeLogs
// orphanRemoval: if a TimeLog is detached from the list, delete it from DB
```

Getters and setters are required — Jackson (the JSON library) uses them to serialize/deserialize objects.

---

### model/TimeLog.java

```java
@ManyToOne(fetch = FetchType.EAGER)
@JoinColumn(name = "task_id", nullable = false)
private Task task;
```

- `@ManyToOne` — many TimeLogs belong to one Task
- `@JoinColumn(name="task_id")` — creates `task_id` foreign key column in MySQL
- `FetchType.EAGER` — load the parent Task immediately when loading a TimeLog.
  LAZY loading caused a serialization error because Jackson tried to access the Task after the DB session was closed.

---

### model/TaskStatus.java

```java
public enum TaskStatus { TODO, IN_PROGRESS, DONE }
```

An enum is a fixed set of allowed values. The API rejects any value outside this set at the validation layer — you cannot accidentally save `"INVALID"` to the DB.

### repository/ — Spring Data JPA

```java
public interface TaskRepository extends JpaRepository<Task, Long> {}
```

Zero SQL written. Spring auto-generates:

| Method call | SQL generated |
|---|---|
| `save(task)` | `INSERT` or `UPDATE` |
| `findById(id)` | `SELECT WHERE id=?` |
| `findAll()` | `SELECT *` |
| `deleteById(id)` | `DELETE WHERE id=?` (cascades to time_logs) |
| `existsById(id)` | `SELECT COUNT(*)` |

Custom method in `TimeLogRepository`:
```java
List<TimeLog> findByTask_Id(Long taskId);
// Spring reads the method name → generates: SELECT * FROM time_logs WHERE task_id=?
```

---

### service/ — Business Logic

```java
public void deleteTask(Long id) {
    if (!taskRepository.existsById(id)) {
        throw new RuntimeException("Task not found with id: " + id);
    }
    taskRepository.deleteById(id);
    // JPA cascade automatically deletes all TimeLogs for this task
}
```

The service owns business rules:
- Validate that a record exists before deleting
- Throw meaningful errors that the GlobalExceptionHandler converts to JSON

### controller/ — REST API

```java
@RestController           // handles HTTP, returns JSON
@RequestMapping("/tasks") // base path for all methods
@CrossOrigin(origins = "http://localhost:3000") // allow React on port 3000

@PostMapping              // POST /tasks
@GetMapping               // GET /tasks
@PutMapping("/{id}")      // PUT /tasks/5
@DeleteMapping("/{id}")   // DELETE /tasks/5

@Valid @RequestBody Task task  // deserialize JSON → Java object, run validation
@PathVariable Long id          // extract {id} from the URL path
@RequestParam Long taskId      // extract ?taskId=5 from the query string
```

HTTP status codes used:
- `ResponseEntity.ok(data)` → `200 OK`
- `ResponseEntity.noContent().build()` → `204 No Content` (standard for DELETE)
- `ResponseEntity.status(400).body(errors)` → `400 Bad Request` (validation failure)
- `ResponseEntity.status(404).body(error)` → `404 Not Found` (record missing)

### config/SecurityConfig.java

```java
.csrf(csrf -> csrf.disable())
// CSRF protection is for browser form submissions.
// Since we use HTTP Basic with JSON APIs, it is not needed.

.authorizeHttpRequests(auth -> auth.anyRequest().authenticated())
// Every endpoint requires authentication. No anonymous access.

.httpBasic(withDefaults())
// Client sends: Authorization: Basic base64(username:password)
```

```java
new BCryptPasswordEncoder()
// Passwords are hashed one-way. Even if the DB leaks, plain passwords are safe.

new InMemoryUserDetailsManager(user)
// Users stored in memory (no DB table). Sufficient for v0.1.
// Will move to a users table in v0.2.
```

---

### config/GlobalExceptionHandler.java

```java
@RestControllerAdvice  // intercepts exceptions from any controller globally

@ExceptionHandler(MethodArgumentNotValidException.class)
// Triggered when @Valid fails
// Returns: { "title": "Title is required", "status": "Status is required" }  →  400

@ExceptionHandler(RuntimeException.class)
// Triggered when service throws RuntimeException("Task not found...")
// Returns: { "error": "Task not found with id: 99" }  →  404
```

Without this, Spring returns raw HTML error pages. This gives the frontend structured, readable JSON.

## Frontend — React

### Core Concepts

**`useState`** — stores values that trigger a re-render when changed
```js
const [tasks, setTasks] = useState([]);
// tasks = current value
// setTasks(newValue) = update value + re-render the component
```

**`useEffect`** — run code when a component mounts (appears on screen)
```js
useEffect(() => {
  fetchTasks();
}, []); // [] = run once on mount only
```

---

### React Router — Client-Side Navigation

```js
<BrowserRouter>
  <Route path="/tasks" element={<PrivateRoute><TaskList /></PrivateRoute>} />
```

- Changes the URL without a full page reload
- `PrivateRoute` checks localStorage for a token — redirects to `/` if missing (frontend route guarding)

```js
const navigate = useNavigate();
navigate('/tasks'); // programmatically change the URL
```

---

### api/axios.js — HTTP Client with Auth Interceptor

```js
const api = axios.create({ baseURL: 'http://localhost:8080' });

api.interceptors.request.use((config) => {
  config.headers = { ...config.headers, ...getAuthHeader() };
  return config;
});
```

The interceptor runs before every request and automatically injects `Authorization: Basic ...`. Every page gets authentication without repeating the header logic.

---

### pages/LoginPage.js — Authentication Flow

```js
const token = btoa(`${username}:${password}`);
// btoa() = Base64 encode. "mohammed:1234" → "bW9oYW1tZWQ6MTIzNA=="
// HTTP Basic auth spec requires base64-encoded credentials

const response = await fetch('http://localhost:8080/tasks', {
  headers: { Authorization: `Basic ${token}` },
});
// Test credentials with a real API call
// 200 → valid → store token | 401 → wrong credentials

localStorage.setItem('tt_token', token);
// Persists across page refreshes until logout or browser clear
```

### Error Parsing (AddTask, EditTask, LogTime)

```js
const parseError = (err) => {
  const data = err?.response?.data;
  if (!data) return 'Cannot reach server. Is the backend running?';
  if (typeof data === 'object') return Object.values(data).join(', ');
  return String(data);
};
```

- `err?.response?.data` — optional chaining: if `response` is undefined (network error), returns undefined instead of crashing
- Handles validation error maps `{ "title": "Title is required" }` → joins values into a readable string

## End-to-End Data Flow — Create a Task

```
1. User fills AddTask form, clicks "Create Task"
2. handleSubmit() → api.post('/tasks', { title, description, status })
3. Axios interceptor adds Authorization: Basic ... header
4. HTTP POST sent to http://localhost:8080/tasks
5. Spring Security validates credentials → allows through
6. TaskController.createTask() receives JSON body
7. @Valid runs validation on the Task object
8.   If invalid → GlobalExceptionHandler → 400 { "title": "Title is required" }
9.   If valid → TaskService.createTask() → TaskRepository.save(task)
10. Hibernate → INSERT INTO tasks (title, description, status) VALUES (...)
11. MySQL executes, returns auto-generated id
12. Spring → 200 { "id": 1, "title": "...", "status": "TODO" }
13. React navigate('/tasks') → TaskList re-fetches → new task appears
```

## MySQL — What Hibernate Generated

| Java | SQL equivalent |
|---|---|
| `@Entity @Table(name="tasks")` | `CREATE TABLE tasks (...)` |
| `@Id @GeneratedValue` | `id BIGINT AUTO_INCREMENT PRIMARY KEY` |
| `@ManyToOne @JoinColumn(name="task_id")` | `task_id BIGINT REFERENCES tasks(id)` |
| `@Enumerated(EnumType.STRING)` | `status VARCHAR(255)` |
| `taskRepository.save(task)` | `INSERT` or `UPDATE` |
| `taskRepository.deleteById(id)` | `DELETE` (cascade deletes time_logs) |

## Key Concepts — Quick Reference

| Concept | Meaning |
|---|---|
| REST API | HTTP endpoints returning JSON. Each URL = a resource (`/tasks`, `/time-logs`) |
| JPA / ORM | Maps Java classes ↔ DB tables. No manual SQL |
| Spring Security | Guards every endpoint. Forces login before any data access |
| BCrypt | One-way password hashing. Safe even if DB is stolen |
| CORS | Browser blocks cross-origin calls. `@CrossOrigin` on controllers allows port 3000 to call port 8080 |
| CSRF | Attack using forged form submissions. Disabled here because we use token-based HTTP Basic |
| useState | React state — changing it triggers a re-render |
| useEffect | React lifecycle — runs after the component mounts |
| Interceptor | Middleware on every HTTP request — used to inject auth headers |
| localStorage | Browser key-value store — persists login token across refreshes |
| Cascade delete | JPA propagates delete from Task (parent) to TimeLogs (children) |
| Enum | Fixed allowed values — prevents invalid data at the type level |
| PrivateRoute | Frontend wrapper that blocks unauthenticated access to pages |
| `@Valid` | Triggers Bean Validation on a request body before the controller method runs |
| `@RestControllerAdvice` | Centralized exception handler for all controllers |
| `FetchType.EAGER` | Load related entity immediately (vs LAZY = only when accessed) |
